Animation by cost minimization

Cost Function

The animation is achieved by using the gradient-descent algorithm to solve the following minimization problem:

(1)Φ^=argΦ⁡minC(Φ),

where the cost function C is defined by:

(2)C(Φ)=∥e(Φ)−g∥⏟goal attraction+∑i=1nFR(∥e(Φ)−oi∥)⏟obstacle-avoidance penalty+∑j=13L(ϕj)⏟Joint-range limit.

Here, g=(xg,yg,zg)T is the goal location, oi is the location of obstacle i. Function e(Φ) computes the arm's forward kinematics and returns the location of the arm's tip e=(ex,ey,ez)T, i.e., the end-effector, given the arm's joint angles, Φ=(ϕ1,ϕ2,ϕ3,ϕ4)T.

Gradient-descent solution for the animation

Algorithm 1

  1. Φt←(ϕ1,ϕ2,ϕ3,ϕ4)T                             ▹ Current joint-angle configuration (i.e., current pose)

  2. et←(ex,ey,ez)T                                      ▹ Current position of the end-effector

  3. g←(gx,gy,gz)T                                       ▹ Location of the goal (or target)

  4. d←0.01                                                     ▹ Acceptable distance to the goal

  5. λ←0.1                                                        ▹ Step of the gradient descent

  6. while ∥et−g∥>d do:

  7. Φt+1←Φt−λ∇C(Φt)                      ▹ Calculate the new joint-angle configuration

  8. drawRobotArm(Φt+1)                            ▹ Draw robot arm at the new configuration

  9. et←e(Φt+1)                                        ▹ Updated end-effector location

  10. Φt←Φt+1                                             ▹ Set current configuration to new configuration

  11. end while

Gradient calculation

(3)∇C(Φt)=∂C(Φt)∂Φt≈[ΔCΔϕ1ΔCΔϕ2ΔCΔϕ3ΔCΔϕ4]=[C([ϕ1ϕ2ϕ3ϕ4]+[Δϕ1000])−C([ϕ1ϕ2ϕ3ϕ4])Δϕ1C([ϕ1ϕ2ϕ3ϕ4]+[0Δϕ200])−C([ϕ1ϕ2ϕ3ϕ4])Δϕ2C([ϕ1ϕ2ϕ3ϕ4]+[00Δϕ30])−C([ϕ1ϕ2ϕ3ϕ4])Δϕ3C([ϕ1ϕ2ϕ3ϕ4]+[000Δϕ4])−C([ϕ1ϕ2ϕ3ϕ4])Δϕ4].

Numerical example

Here is a numerical example showing the calculation of the gradient of the cost function for a given join-angle configuration. Assume that the current robot arm joint-angle configuration is Φt=(10,5,−15,20)T and that we set the incremental variations Δϕi=0.05. Then, the gradient vector calculation is:

(4)∇C(Φt)=∂C(Φt)∂Φt≈[ΔCΔϕ1ΔCΔϕ2ΔCΔϕ3ΔCΔϕ4]=[C([105−1520]+[0.05000])−C([105−1520])0.05C([105−1520]+[00.0500])−C([105−1520])0.05C([105−1520]+[000.050])−C([105−1520])0.05C([105−1520]+[0000.05])−C([105−1520])0.05].

Of course, the actual values of the cost, i.e., C(⋅) depend on forward-kinematics function that returns that location of the end-effector, i.e., e(Φ). The forward-kinematics function is specific for each robot-arm. Also, depending on how the input angles are processed inside the cost function calculation (e.g., rotations that use sine and cosine functions, we will need to convert the values of the angles from degrees to radians before we pass them to the forward-kinematics function.
